初始化

CountDownLatch是通过一个计数器(这个计数器实际就是AQS的同步状态变量state)来实现它的功能的，当我们在new 一个CountDownLatch对象的时候需要带入该计数器值，该值就表示了线程的数量。每当一个线程完成自己的任务后，计数器的值就会减1。当计数器的值变为0时，就表示所有的线程均已经完成了任务，然后就可以恢复先前我们让等待的线程继续执行了。

总结

CountDownLatch是共享锁的一种，内置一个计数器(AQS的state变量)，当我们新建一个CountDownLatch实例时，传递一个变量值进去，这个值是计数器的初始值，构造方法会去检查state是否为0，不然会一直自旋等待，接着调用await()方法阻塞我们想要阻塞的线程，await方法会自旋的检查state变量是否为0，为0的时候就唤醒被阻塞的线程，另一方面await调用后阻塞阶段，接着调用countDown减少计数，countDown方法会先检查一遍state是否已为0，接着对state变量-1。
